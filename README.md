## 开发规范

### Git 工作流程（半自动化）
本项目采用 AI 辅助的 Git 半自动化管理：

1. **初始化阶段**
   - 设置 Git 仓库
   - 创建 .gitignore 文件
   - 配置基本提交模板

2. **日常开发流程**
   - 完成功能开发后，我会检查代码变更
   - 我会向您说明准备提交的内容和原因
   - 执行 `git add` 和 `git commit`，并附上清晰的 commit message
   - 向您报告提交结果

3. **提交规则**
   - ✅ 每个提交都是完整的功能单元
   - ✅ Commit message 说明"为什么改"而不仅仅是"改了什么"
   - ✅ 提交前会向您说明并获确认
   - ❌ 不会擅自提交，除非您明确要求

4. **项目状态跟踪**
   - 定期向您报告当前分支、未提交的变更
   - 重大里程碑时会总结项目进展
