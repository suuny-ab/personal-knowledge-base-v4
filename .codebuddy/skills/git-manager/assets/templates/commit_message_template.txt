# 提交类型：
# feat: 新功能
# fix: Bug 修复
# docs: 文档变更
# style: 代码格式（不影响代码运行）
# refactor: 重构（既不是新增功能，也不是修复bug）
# perf: 性能优化
# test: 增加测试
# chore: 构建过程或辅助工具的变动
# revert: 回滚之前的提交

# 格式：
<类型>(<范围>): <主题>

<详细描述>

<脚注>

# 示例：
# feat(auth): 添加用户登录功能
#
# - 实现邮箱/密码登录
# - 集成 JWT token 认证
# - 添加登录表单验证
#
